# Spot the Fake Photo

Given one image, decide if it's a **real photo** or a **photo of a screen** (someone re-photographing a phone/laptop/printout instead of the real thing).

## Results
- **Accuracy:** 92.0% ± 6.8% (5-fold CV, 100-image dataset).
- **Latency:** ~113 ms/image (112.76 ms median), laptop CPU.
- **Cost:** free on-device; ~$1.50 per million images (~$0.0015/1,000) on a cheap cloud CPU instance, if ever needed.

Full write-up: **[NOTE.md](./NOTE.md)**.

## How it works
7 hand-crafted features per image (FFT peak energy, Laplacian variance, saturation skew/p95, LBP entropy, DCT block periodicity, highlight ratio) → `StandardScaler -> LogisticRegression`.

## Files
| File | What it does |
|---|---|
| `features.py` | Extracts the 7 features from an image. |
| `train.py` | Loads `real/` and `screen/`, extracts features, runs 5-fold CV, fits and saves `model.pkl`. |
| `predict.py` | `python predict.py image.jpg` → prints a score between 0–1 (a number from 0 to 1: 0 = real photo, 1 = photo of a screen) and logs latency to stderr. |
| `model.pkl` | Trained logistic regression pipeline (generated by `train.py`). |
| `NOTE.md` | Approach, accuracy, latency/cost, trade-offs, and what I'd improve. |

## Setup
```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

## Train
Put ~50 photos in `real/` and ~50 in `screen/`, then:
```bash
python train.py
```
This prints CV accuracy, feature weights, and writes `model.pkl`.

## Predict
```bash
python predict.py some_image.jpg
```
